# Full Curriculum Plan

This plan turns the project catalog into a coherent learning path. The goal is not to finish every project. The goal is to build enough serious systems that the learner develops Staff-level instincts: invariants, failure modes, debugging, operations, and tradeoffs.

The detailed plan is split into roadmap chapters so the curriculum can be read like a book and maintained without one giant document.

## Roadmap Chapters

| Chapter | What It Covers | When To Read It |
| --- | --- | --- |
| [Chapter 1: Curriculum Shape and Levels](chapters/01-curriculum-shape-and-levels.md) | This chapter explains the learning loop and the level-by-level progression from tool fluency to AI systems and capstones. | Start here when orienting a new learner or explaining the full arc. |
| [Chapter 2: Concepts and Project Tracks](chapters/02-concepts-and-tracks.md) | This chapter is the expansion map: concepts to teach and tracks that can grow into more implementation ladders. | Use this when expanding the curriculum or deciding what concepts deserve new exercises. |
| [Chapter 3: Recommended Master Sequence](chapters/03-master-sequence.md) | This chapter gives the recommended order for a learner who wants the most coherent Staff-level progression. | Use this when choosing the next large learning phase. |
| [Chapter 4: Scaffolding, Validation, and Publishing](chapters/04-scaffolding-validation-and-publishing.md) | This chapter describes how to turn the curriculum into maintained artifacts: scaffolds, validators, packaging, and references. | Use this when creating, auditing, or releasing curriculum artifacts. |

## Curriculum Loop

| Step | Learner Action | Evidence |
| --- | --- | --- |
| Concept | Read the self-contained concept chapter and name the invariant. | A short explanation in the learner's own words. |
| Exercise | Implement the smallest current milestone. | Focused validator output. |
| Extension | Add a failure mode, scale case, or operational concern. | A test, trace, benchmark, or design note. |
| Review | Explain the tradeoff and remaining risk. | Staff-level review answers. |
